1) Login / Security

Single Admin Login (username + password) — यह मुख्य admin होगा।

Main Admin login के बाद ही पूरा control मिलेगा।

Mini-admin credentials (links, username, password) को देखने/regen करने के लिए फिर से main admin password माँगा जाएगा (extra security).

2) Dashboard (Top-level)

क्या दिखेगा:

Court Basic Info summary (editable):

Court Name

Total Departments

Total Judges (or officers)

Total Buildings

Total Rooms

Quick action buttons:

Add Building

Add Room

Create Mini-Admin

View Unanswered Queries

Recent activity list (latest 5 edits / additions)

Top-most asked questions (short list) — लेकिन यह बार-बार duplicate न दिखे (only frequently asked)

(कोई daily assistant usage stat नहीं होगा)

3) General Info (Basic Court Info)

Purpose: Court का मुख्य विवरण जो admin add/edit कर सके।
Fields:

Court Name

Address / Landmark (text)

Contact Numbers (reception, PRO, legal aid)

Official Email / Website

Working Hours / Lunch Break

Holidays list (editable)
UI: simple form, save/edit, history of changes

4) Building & Location Management (Core)

Note: Coordinates removed — सिर्फ textual details और images रखेंगे.

क्या add कर पाएगा admin:

Building Name

Short Description

Landmark / Directional Notes (text): (उदा. “Near Gate 1, opposite canteen”)

Number of Floors

Upload Building Photo (optional)

Actions: Edit / Delete / View Rooms

UI: Building list/table with Add Building modal

5) Room & Courtroom Management

Important: जब कोई room बनायेगा, उसे building select करना होगा।

Room form fields:

Room Number / Name

Select Building (dropdown)

Floor (dropdown)

Purpose (hearing room, office, counter, canteen etc.)

In-charge (select existing staff or add new)

Timing (e.g., 10:00–16:00)

Status (Open / Closed / In Use)

Photo (optional)

Short Notes (e.g., “File submission counter”, “Document verification”)

Extra behavior:

Room list grouped by Building

Quick search by Room No. or Building

When you add a room, assistant will automatically use this info in replies

6) Departments & Staff (Officers)

Functions:

Add/Edit Departments (Civil, Criminal, Family, Records, Accounts, etc.)

Add/Edit Staff / Officers:

Name, Designation, Department, Assigned Building & Room, Contact, Photo

Assign staff to rooms/departments

UI: Directory table with filters (by dept / building)

7) Mini-Admin (Mini Panel) Management

Purpose: प्रत्येक विभाग / building / officer के लिए एक छोटा admin access देना।

Flow:

Main admin clicks Create Mini-Admin.

Form: Name, Role (dept/building/person), Scope (e.g., Building A / Department X), Email (optional).

System generates:

Mini-panel link (unique URL)

Username

Temporary Password

Main admin can View / Regenerate credentials — परन्तु इसे देखने/regen करने के लिए main admin password दुबारा माँगा जाएगा।

Mini-admin permissions: only CRUD within their scope (e.g., add rooms in their building, edit FAQs for their dept, view tickets for their scope).

Notes:

Mini-panel users login to a mini interface (limited UI): add/edit rooms under their building, update staff for that dept, add FAQs for their area.

Main admin can revoke access anytime.
